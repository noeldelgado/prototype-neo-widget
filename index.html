Avatars list

  • 1st level items list + 1st level event listeners

Vote list

  • Children communication with container to update state via props
  • Each children has two event listeners

Self updated state

  • Each list item will update “itself” via setState
  • The parent.state will be modified to hold that individual state too (using a keyPath-like format)
  • The parent will run a new diff/patch of its tree
  • The children will check if its parent holds its keyPath record and merge it into its current state
